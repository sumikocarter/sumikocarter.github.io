Interaction Design Exercise

#1 Find this garment

Imagine a mobile app that enables you to impulsively identify and purchase a garment or accessory that you see in real life.
Design an end-to-end flow covering the experience from the moment of awareness to purchase completion.

1. Home/Welcome

This is a visual fashion search engine that identifies garments or accessories via four user inputs/actions. 

  1. Enter text or speak

  2. Take a photo

  3. Choose an image from the phone library

  4. Scan a barcode, QR code, or garment tag.

 1a. Menu Open

From the home screen, the user can open a drawer navigation menu revealing administrative items like settings and support. 

2. Search

As the user engages the search box, the search becomes the primary focus in the viewport. We also list the 3 other search methods again as options. If the user has visited this app before, their recent searches will appear below as quick links with the ability to dismiss.  

3. Choose an Existing Photo

The user has decided to choose a photo from their existing camera library. Tool tips about zooming and cropping materialize to improve search result matches.

A fixed navigation target adjusts the height of the tiled photo area.

4. Search Results From Photo

After the user hits “Use Photo” a listing of all possible results appear as tiles. This design embraces large product images to enhance the site experience and help the user recognize the desired product.

In the navigation, the search is still exposed for easy editing and all filter/sort tools are available. The user can change the category; refine based on standard data found on a product detail page; or sort by best match, most popular, or price. 

5. Product Detail

Customers need adequate information about a product before they will buy it online. These product pages use a combination of text and images to show product information, availability, and price. The user can swipe to see the next result listing without returning to the previous screen.

Fixed to the bottom of the viewport is an action menu that exposes all primary shopping actions including selecting a color and size to purchase.

6. Expanded Purchasing Menu on Product Detail

When the user has clicked on the fixed menu, all standard shopping actions are revealed including purchase selections, buy now, add to cart, find used, locate, and share.

7. Expanded Cart Menu

After the user adds an item to their cart the cart icon is updated. The cart menu slides from the right with the pertinent information. The user can then checkout or express purchase with paypal, wallet, or amazon. 

8. Log in to Checkout

After the user has decided to checkout, they are prompted to log in or use another popular method of paying.

The ability to checkout as a guest is provided, but the user will have to follow the long and standard checkout process. By logging in, the customer’s preferred method of payment and shipping are saved and skips the user to review the order.

9. Review Order

The user reviews pre-populated shipping, billing, and product information then places their order.

The main action is fixed to the bottom and is always present in the viewport.

10. Confirmation/Thank You

The user views order confirmation. This page instructs the user to check their email for more information. At the end popular items are displayed to encourage a new search.
